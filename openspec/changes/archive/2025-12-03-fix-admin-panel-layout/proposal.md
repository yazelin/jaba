# Change: 修復管理員面板佈局、菜單上傳與訂單權限問題

## Why
管理員頁面存在多個 UI/UX 與安全問題：
1. 店家管理面板使用收合式設計，最下方的店家經常因為面板高度限制而看不到
2. 菜單上傳選擇「新增店家」時出現錯誤：`Unexpected token '<'`
3. 管理員希望調整面板佈局：左側店家管理、中間對話框、右側資訊面板
4. 拍照新增菜單的 icon 太小不明顯
5. **安全漏洞**：任何用戶可以透過對話刪除其他人的訂單

## What Changes
- **三欄佈局**：左側店家管理、中間對話框、右側資訊面板（今日店家、今日訂單、付款狀態）
- **修復店家管理面板**：調整高度計算，確保所有店家都能滾動查看
- **修復菜單上傳錯誤**：處理中文店名轉換與 API 錯誤回應
- **增強菜單上傳入口**：改為明顯的按鈕樣式並加上文字提示
- **訂單權限檢查**：一般用戶只能取消自己的訂單，管理員可取消任何人訂單

## Impact
- Affected specs: `admin-ui`（新增）、`order-management`（修改）
- Affected code:
  - `templates/manager.html`：HTML 結構調整為三欄
  - `static/css/style.css`：三欄佈局樣式
  - `main.py`：修復 API 錯誤處理
  - `app/claude.py`：加入訂單權限檢查
