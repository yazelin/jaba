## MODIFIED Requirements

### Requirement: 手機版底部留白避免遮擋
訂餐頁和管理員頁在手機版 SHALL 保留足夠的底部空間，避免美食評論區遮擋重要內容。

#### Scenario: 訂餐頁大家的訂單統計可見
- **GIVEN** 使用者在手機瀏覽器開啟訂餐頁
- **AND** 美食評論區為展開狀態
- **WHEN** 使用者查看「大家的訂單」面板
- **THEN** 「共 x 筆」統計行完整顯示，不被美食評論區遮擋

#### Scenario: 管理員頁收款面板完整顯示
- **GIVEN** 管理員在手機瀏覽器開啟管理頁
- **AND** 美食評論區為展開狀態
- **WHEN** 管理員查看收款面板
- **THEN** 收款面板下緣完整顯示，不被美食評論區遮擋

## ADDED Requirements

### Requirement: 返回按鈕明顯化
返回看板按鈕 SHALL 具有明顯的按鈕外觀，讓使用者容易辨識和點擊。

#### Scenario: 返回按鈕視覺設計
- **GIVEN** 使用者在訂餐頁或管理員頁
- **WHEN** 查看頁面左上角
- **THEN** 返回按鈕具有背景色、圓角等按鈕特徵
- **AND** 與周圍元素有足夠對比度

#### Scenario: 返回按鈕互動回饋
- **GIVEN** 使用者將滑鼠移到返回按鈕上
- **WHEN** hover 狀態
- **THEN** 按鈕有視覺變化（如背景色加深）表示可點擊
